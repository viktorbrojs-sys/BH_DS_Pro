# Автоэнкодер для сжатия текстовых данных

Автоэнкодер для сжатия текстовых данных с малым размером скрытого представления: Разработка автоэнкодера, который эффективно сжимает текстовые данные, используя компактное скрытое представление для уменьшения объёма хранения.

---
## Задача

Обучить автоэнкодер на корпусе новостных текстов и сравнить качество реконструкции при разных размерах латентного пространства:

- `latent_dim=5` — агрессивное сжатие (1000 → 5)
- `latent_dim=50` — умеренное сжатие (1000 → 50)

---
## Данные

- **Корпус**: Reuters (NLTK) — новостные тексты на английском языке
		- **Количество документов**: Всего 10,788 новостных статей, общим объемом около **1.3 миллиона слов**
		- **Тематика**: Статьи охватывают финансовые и экономические темы (торговля, сырьевые рынки, авиация и т.д.).
- **Объём выборки**: 1000 случайных документов (`random.seed(42)`)
- **Векторизация**: TF-IDF (`max_features=1000`, `min_df=3`, `max_df=0.8`)
- **Нормализация**: `MaxAbsScaler` (сохраняет нули разреженной матрицы)

---
## Архитектура модели

```
Энкодер:  1000 → 512 → 128 → latent_dim
Декодер:  latent_dim → 128 → 512 → 1000
```

Каждый слой содержит: `Linear → BatchNorm1d → ReLU → Dropout(0.2)`

---
## Гиперпараметры обучения

| Параметр          | Значение                                   |
| ----------------- | ------------------------------------------ |
| Epochs            | 150                                        |
| Batch size        | 32                                         |
| Learning rate     | 5e-4                                       |
| Optimizer         | Adam (weight_decay=1e-5)                   |
| LR Scheduler      | ReduceLROnPlateau (patience=7, factor=0.5) |
| Early stopping    | patience=15                                |
| Gradient clipping | max_norm=1.0                               |
| Dropout           | 0.2                                        |

---
## Метрики качества

Основная метрика — **cosine similarity** между оригинальным и восстановленным вектором.  
MSE используется как функция потерь при обучении, но не является показательной метрикой для оценки качества, так как TF-IDF векторы разрежены (~95% нулей).

## Визуализации

1. **График потерь** — сравнение кривых MSE для `latent_dim=5` и `latent_dim=50`
2. **Латентное пространство** — scatter plot первых двух координат латентных векторов (первые 100 документов)
3. **Топ-слова** — сравнение ключевых слов оригинального и восстановленного текста
4. **Гистограмма cosine similarity** — распределение качества реконструкции по всем документам

## Установка и запуск

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Скачать корпус Reuters

```python
import nltk
```
