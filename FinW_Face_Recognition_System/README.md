# Система видеонаблюдения с распознаванием лиц

Python-проект для видеонаблюдения с детекцией людей через YOLOv8, распознаванием лиц и уведомлениями в Telegram.

## Что умеет система

- Детектирует людей в видеопотоке с помощью YOLOv8
- Определяет, пересёк ли человек заданную зону (полигон на экране)
- Находит лицо у каждого человека в зоне через OpenCV DNN (SSD ResNet)
- Сравнивает лицо с базой известных людей
- Отправляет фото лица и карточку человека в Telegram
- Сохраняет фото неизвестных для последующей разметки
- Ведёт журнал событий в SQLite
- Предоставляет REST API (FastAPI) и веб-дашборд (Streamlit)
- Работает на CPU без GPU (с опциональной поддержкой CUDA)

## Установка

### Быстрый старт

```bash
git clone https://github.com/your-username/face-recognition-system.git
cd face_recognition_system
python setup.py
```

`setup.py` сделает всё автоматически: создаст папки, установит зависимости, скачает модель детектора лиц и инициализирует базу данных.

### Ручная установка

```bash
pip install -r requirements.txt
```

Для работы на CPU (без CUDA) PyTorch устанавливается так:

```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
```

## Настройка

Создайте файл `.env` в папке проекта или задайте переменные окружения:

```env
TELEGRAM_BOT_TOKEN=your_token_here
TELEGRAM_CHAT_ID=your_chat_id_here

# Источник видео: 0 = веб-камера, или путь к файлу
CAMERA_SOURCE=0

# Зона наблюдения — полигон [[x,y], ...]
ZONE_POLYGON=[[100,200],[300,200],[300,400],[100,400]]

# Порог совпадения лиц (0.0 — строго, 1.0 — мягко)
FACE_MATCH_THRESHOLD=0.6

# Обрабатывать каждый N-й кадр
PROCESS_EVERY_N_FRAMES=5

# Использовать GPU
USE_GPU=False

# Пауза между уведомлениями об одном человеке (секунды)
DEBOUNCE_SECONDS=5
```

### Получение Telegram Bot Token

1. Напишите боту [@BotFather](https://t.me/BotFather) команду `/newbot`
2. Следуйте инструкциям, получите токен
3. Узнайте свой chat_id через [@userinfobot](https://t.me/userinfobot)

## Добавление известных лиц

### Вариант 1 — пакетное добавление из папки

Создайте папку с именем человека внутри `known_faces/`. Имя папки = имя человека, пробелы заменяются подчёркиваниями. Внутри положите любое количество фото.

```
known_faces/
├── Ivan_Petrov/
│   ├── front.jpg        # анфас
│   ├── side.jpg         # профиль
│   └── glasses.jpg      # в очках
├── Anna_Sidorova/
│   ├── photo1.jpg
│   └── photo2.jpg
└── Aleksey_Ivanov/
    └── office.jpg
```

**Советы по фото:**
- Хорошее освещение, лицо чёткое, не смазанное
- Разные ракурсы (анфас, 3/4, профиль) улучшают распознавание
- Фото из реальной обстановки лучше, чем идеальные студийные снимки
- 3–5 фото на человека — оптимально

Запустите обучение:

```bash
python train_faces.py
```

Вывод после обучения:
```
Результат:
  Людей добавлено:  3
  Фото добавлено:   7
  Пропущено:        0  (уже были в базе)
  Ошибок:           0  (файл не читается или лицо не найдено)
```
Если человек уже есть в базе — его папка пропускается. Чтобы переобучить — удалите запись из базы и запустите снова.

## Запуск

### Основная система

```bash
python main.py
```

Для тестирования на видеофайле задайте `CAMERA_SOURCE=test_video.mp4`.

На экране отображается:
- Жёлтый полигон — зона наблюдения
- Зелёный bounding box — человек вне зоны
- Красный bounding box — человек в зоне
- Синий прямоугольник — найденное лицо
- Надпись KNOWN/UNKNOWN над лицом
- Текущий FPS

Нажмите **Q** для остановки.

### REST API

```bash
python api_server.py
```

Эндпоинты:

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/events` | Последние события |
| GET | `/api/events/unknown` | Только неизвестные |
| GET | `/api/stats` | Сводная статистика |
| GET | `/api/health` | Проверка работы сервера |

Документация API автоматически доступна по адресу `http://localhost:8500/docs`.

### Веб-дашборд

```bash
streamlit run streamlit_app.py
```

Открывается в браузере. Показывает:
- Счётчики событий
- Временной график активности
- Частоту появления людей
- Таблицу последних событий

### Проверка камеры

```bash
python test_camera.py          # проверяет CAMERA_SOURCE из .env
python test_camera.py video.mp4  # проверяет конкретный файл
```

### Текстовый отчёт по метрикам

```bash
python eda.py
```

## Архитектура

```
Камера/видео
    ↓
[Поток 1: video_capture_thread]
    ↓ queue (frame_queue)
[Поток 2: face_processing_thread]
    ├── YOLO → детекция людей
    ├── Проверка зоны (полигон)
    ├── OpenCV DNN → детекция лиц
    ├── Гистограммный эмбеддинг
    ├── Сравнение с базой (косинусное расстояние)
    ├── Антиспам (debounce по tracker_id)
    └── Telegram / SQLite / сохранение фото
    ↓ queue (result_queue)
[Основной поток: отрисовка + FPS]
```

## Технологии

| Компонент | Технология |
|-----------|------------|
| Детекция людей | YOLOv8s (ultralytics) |
| Детекция лиц | OpenCV DNN (SSD ResNet-10) |
| Эмбеддинги лиц | Гистограммный дескриптор (HSV + Lab + Gray) |
| Трекинг | Центроидный трекер (собственная реализация) |
| База данных | SQLite (через стандартный sqlite3) |
| Telegram | python-telegram-bot v20+ (async API) |
| REST API | FastAPI + uvicorn |
| Дашборд | Streamlit |
| Многопоточность | threading + queue |

## Ограничения
- **Точность распознавания** — гистограммный эмбеддинг чувствителен к освещению и повороту головы. 
При переходе на GPU-сервер лучше использовать InsightFace или DeepFace.

## Требования

- Python 3.9+
- CPU с поддержкой AVX (для OpenCV DNN)
- Для GPU: CUDA 11.8+ и соответствующая версия PyTorch
