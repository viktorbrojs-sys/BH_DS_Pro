# Детекция касок на строительных площадках `helmet-detection` (YOLOv5)

Система компьютерного зрения для контроля техники безопасности: определяет, надета ли каска (класс **helmet**) или голова открыта (класс **head** = нарушение), в режиме реального времени.

---
## Структура проекта

| setup.py         | установка окружения, клонирование YOLOv5                 |
| ---------------- | -------------------------------------------------------- |
| eda.py           | разведочный анализ датасета (EDA)                        |
| train.py         | обучение модели                                          |
| metrics.py       | просмотр метрик после обучения                           |
| detect_rt.py     | детекция в реальном времени / видеофайл                  |
|                  |                                                          |
| requirements.txt | список Python-пакетов, необходимых для работы проекта    |
|                  |                                                          |
| dataset/         | сюда кладём датасет                                      |
| dataset_mini/    | создаётся автоматически, если нужен маленький датасет    |
| runs/            | результаты обучения (создаётся автоматически)            |
| outputs/         | сохранение обработанных фото и видео                     |
| outputs/plots    | графики-результаты EDA и обучения модели                 |
| test/            | необязательная, создаётся вручную и здесь хранятся тесты |
| yolov5/          | создаётся автоматически (после запуска setup.py)         |

## Быстрый старт

### 1. Установка окружения

```bash
python setup.py
```

Скрипт:
- клонирует `github.com/ultralytics/yolov5` в папку `yolov5/`
- устанавливает все зависимости
- создаёт структуру папок `dataset/` и `outputs/`

### 2. Подготовка датасета

**Датасет:** Hard Hat Workers (Roboflow)  
**Ссылка:** https://universe.roboflow.com/yolo-whioc/hard-hat-workers-laqdp
Описание: dataset/README.roboflow.txt
Инструкция по подготовке датасета Hard Hat Workers: dataset/README_dataset.txt

Скачайте датасет в формате **YOLOv5 PyTorch** (получите ZIP-архив).

**Структура после распаковки:**
```
dataset/
	data.yaml
	images/
		train/
		val/
		test/       (опционально)
	labels/
	    train/
	    val/
	    test/       (опционально)
```

Поместите содержимое архива в папку `dataset/` так, чтобы файл `dataset/data.yaml` существовал.

**Классы датасета:**
- `head` — голова без каски (нарушение ТБ!)
- `helmet` — каска надета (норма)

### 3. Анализ датасета (EDA)

```bash
python eda.py
```

Создаёт графики в `outputs/plots/`:
- Примеры изображений с bounding boxes (train/val)
- Распределение классов по выборкам
- Гистограмма размеров объектов

### 4. Обучение модели

```bash
# Стандартный запуск (50 эпох, batch=16, модель yolov5m)
python train.py

# Кастомные параметры
python train.py --epochs 100 --batch 32 --model yolov5s.pt

# Только CPU, меньше эпох для теста
python train.py --epochs 10 --batch 4
```

| Параметр     | По умолчанию            | Кастомные (пример)                  | Описание                      |
| ------------ | ----------------------- | ----------------------------------- | ----------------------------- |
| `--img`      | 640                     | 160                                 | Размер входного изображения   |
| `--batch`    | 16                      | 4                                   | Размер батча (-1 = автовыбор) |
| `--epochs`   | 50                      | 25                                  | Количество эпох               |
| `--weights`  | yolov5m.pt              | yolov5s.pt                          | Модель: s/m/l/x               |
| `--cache`    | False                   | False                               | Кэш датасета в RAM            |
| `--fraction` | 1                (100%) | 0.1                           (10%) | Объём датасета для обучения   |

**Модели по скорости/точности:**

| Модель  | Размер  | Скорость | Точность |
| ------- | ------- | -------- | -------- |
| yolov5s | малая   | быстро   | ниже     |
| yolov5m | средняя | средне   | хорошо   |
| yolov5l | большая | медленно | отлично  |
Результаты сохраняются в `runs/train/helmet_<дата>_<время>/`.

### 5. Просмотр метрик

```bash
python metrics.py
```

Показывает и сохраняет:
- Кривые mAP, Precision, Recall, Loss по эпохам
- Confusion Matrix
- PR-кривую, F1-кривую

### 6. Детекция в реальном времени

```bash
# Веб-камера
python detect_rt.py

# Видеофайл
python detect_rt.py --source path/to/video.mp4

# Изображение
python detect_rt.py --source path/to/image.jpg

# Свои веса (после обучения)
python detect_rt.py --weights runs/train/helmet_xxx/weights/best.pt
```

| Параметр     | По умолчанию | Описание                  |
| ------------ | ------------ | ------------------------- |
| `--source`   | 0            | Источник (0=камера, файл) |
| `--weights`  | авто         | Путь к весам              |
| `--conf`     | 0.4          | Порог уверенности         |
| `--iou`      | 0.45         | Порог IoU для NMS         |
| `--img-size` | 640          | Размер кадра              |
| `--device`   | авто         | cpu / 0 (GPU)             |
Нажмите **q** для выхода. Видео сохраняется в `outputs/`.

## Алерты

При обнаружении головы без каски:
- Бокс выделяется **красным**
- В нижней части экрана появляется алерт `! NO HELMET !`

При наличии каски:
- Бокс выделяется **зелёным**

Каждый объект получает уникальный **ID трекера** (сохраняется между кадрами).

## Возможные проблемы

**Нет GPU:**  
Скрипты автоматически переключаются на CPU. Обучение будет медленнее, 
уменьшите `--batch` до 4–8.

**Нет камеры:** 
`detect_rt.py` автоматически переключится на `headless` - будет писать в лог. 
Используйте `--source video.mp4` с видеофайлом.

**data.yaml не найден:**  
Убедитесь, что датасет распакован в `dataset/` и файл `dataset/data.yaml` существует.

**Ошибки зависимостей:**  
```bash
pip install -r requirements.txt
pip install -r ./yolov5/requirements.txt
```

## Дополнительно

- Для точного трекинга рекомендуется заменить `SimpleTracker` на [SORT](https://github.com/abewley/sort) или [ByteTrack](https://github.com/ifzhang/ByteTrack)
- Для ускорения инференса можно экспортировать модель в ONNX:
	`python yolov5/export.py --weights best.pt --include onnx`
- Все графики сохраняются в `outputs/plots/` в формате PNG.

### **Рекомендации для быстрого теста работоспособности:**

- *Для быстрого обучения на CPU (на слабом ПК):*

*`   python train.py --fraction 0.1 --img 160 --weights yolov5s.pt --epochs 10   `*

*`   python train.py --fraction 0.2 --img 320 --weights yolov5m.pt --epochs 30   `*
